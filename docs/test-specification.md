## 1. テスト戦略

### 1.1 テストレベル別の方針

| レベル | 目的 | ツール | 実行タイミング |
|--------|------|--------|--------------|
| 単体テスト | 関数・コンポーネントの独立した動作確認 | Jest + React Testing Library | PR作成時 |
| 統合テスト | APIエンドポイントの動作確認 (実DB使用) | Jest + Supertest + SQLite | PR作成時 |
| E2Eテスト | 画面操作の正常フロー確認 | Playwright | リリース前 |

### 1.2 テスト優先度

- **高**: API バリデーション、認証・認可、ステータス遷移
- **中**: フォームバリデーション、データ表示、フィルター・ソート
- **低**: UIレイアウト、スタイリング

# パッケージ (要インストール)

**設定ファイル**: `jest.config.ts`

```ts
export default {
  preset: 'ts-jest',
  testEnvironment: 'node',        // API テスト
  // testEnvironment: 'jsdom',   // コンポーネントテスト
  setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],
};
```

**テスト前準備**:
```bash
# テスト用DBのマイグレーション
DATABASE_URL="file:./test.db" npx prisma migrate deploy

# テストデータのシード
npx ts-node prisma/seed-test.ts
```

export default {
